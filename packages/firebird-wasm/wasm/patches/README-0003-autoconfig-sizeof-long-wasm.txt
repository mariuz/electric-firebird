SIZEOF_LONG patch for 32-bit WASM
===================================

autoconfig.h is generated by the native CMake configure step on a 64-bit
host where sizeof(long)==8.  Emscripten targets 32-bit WASM where
sizeof(long)==4.  Without this fix, fb_types.h defines SLONG as `int`
(due to SIZEOF_LONG==8) while types_pub.h (which doesn't see _LP64)
uses `signed long`, causing a "typedef redefinition with different types"
error.

This is NOT a git-format-patch because the target file (autoconfig.h) is
generated, not committed.  The fix is applied via sed in build.sh after
the native CMake configure step:

    sed -i 's/^#define SIZEOF_LONG[[:space:]]*8/#define SIZEOF_LONG 4/' \
        "${FIREBIRD_SRC}/src/include/gen/autoconfig.h"

The substitution is idempotent (no-op if already patched to 4).
